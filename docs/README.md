# OctoAcme Project Management Docs

This folder contains the OctoAcme project management process documents. The README below provides a short summary of our approach and direct links to each document for easy discovery.

## Quick Summary of Project Management Processes

OctoAcme operates through a structured, customer-first approach that emphasizes iterative delivery, clear ownership, and continuous learning:

- **Initiation**: Capture project intent with a one-pager, align stakeholders, and decide go/no-go for planning.
- **Planning**: Create a prioritized backlog, estimate scope, identify dependencies and risks, and build a release plan.
- **Execution & Tracking**: Use project boards and a PR workflow with CI checks; run daily standups (15 min) and weekly delivery syncs to surface blockers and progress.
- **Release & Deployment**: Follow pre-release checks, automated pipelines for staging/production, and post-deploy verification and communications.
- **Retrospective & Continuous Improvement**: Run retros, capture action items, and track improvements in the backlog.

### Key Principles

- **Customer-first**: Prioritize customer value and usability.
- **Iterative delivery**: Deliver small, testable increments.
- **Clear ownership**: Each project has a named Project Manager and Product Lead.
- **Data-informed decisions**: Measure impact and iterate based on evidence.
- **Psychological safety**: Encourage feedback and learning.

### Core Roles

- **Developers**: Implement features, collaborate on design/code reviews, help identify technical risks.
- **Product Managers**: Define vision and success metrics, prioritize backlog, validate solutions.
- **Project Managers**: Coordinate delivery, manage schedules/risks/dependencies, facilitate meetings and documentation.

### Communication Cadence

- Weekly sync between PM + PdM
- Twice-weekly standups for delivery team (or as agreed)
- Monthly stakeholder updates
- Ad-hoc escalations as needed
